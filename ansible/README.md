# Ansible — Configuration & Deployment

Configures EC2 instances provisioned by Terraform. Inventory files are generated automatically by the `compute` Terraform module into `inventory/`.

## Roles

| Role | Purpose |
|------|---------|
| `vernemq` | Installs and configures VerneMQ (supports `release`, `source`, and `git_clone` build modes) |
| `bench` | Installs [emqtt-bench](https://github.com/emqx/emqtt-bench) and node_exporter |
| `monitoring` | Installs Prometheus, Grafana (HTTPS with self-signed cert), and node_exporter |

## Playbooks

| Playbook | Purpose |
|----------|---------|
| `deploy_vernemq.yml` | Install VerneMQ on cluster nodes |
| `deploy_bench.yml` | Install emqtt-bench on bench nodes |
| `deploy_monitoring.yml` | Install Prometheus + Grafana on monitor node |
| `configure_cluster.yml` | Form the VerneMQ cluster and verify health |
| `restart_cluster.yml` | Lightweight cluster reset between scenarios |
| `teardown_cluster.yml` | Stop services and clean up node state |

## Inventory

Inventory is auto-generated by Terraform into `inventory/hosts` (or `inventory/hosts.<cluster-id>` for workspaces). It contains three groups:

- `[vmq_nodes]` — VerneMQ cluster nodes
- `[bench_nodes]` — Benchmark client nodes
- `[monitor]` — Prometheus/Grafana node

## VerneMQ Build Modes

The `vernemq` role supports three build modes via the `build_mode` variable:

| Mode | Description |
|------|-------------|
| `release` | Download a pre-built release package |
| `source` | Build from a local source tarball |
| `git_clone` | Clone a git repo at a specific ref and build from source (used by `run_benchmark.sh`) |

## Key Configuration

The `vernemq` role exposes 30+ tunables in `roles/vernemq/defaults/main.yml` including:

- MQTT settings: `listener_port`, `max_connections`, `acceptors`
- Cluster settings: `cluster_listener_port`, `gossip_interval`
- Balance/rebalance: `balance_enabled`, `rebalance_enabled`, thresholds
- Health: `cluster_health_quorum`, `dead_node_cleanup_timeout`
- Performance: `connection_pool_size`, `reg_trie_workers`, `async_threads`

These can be overridden via tuning profiles (see `profiles/`).

## Running Manually

```bash
# Deploy VerneMQ from a git repo
ansible-playbook deploy_vernemq.yml \
    -e build_mode=git_clone \
    -e vernemq_git_repo=https://github.com/vernemq/vernemq.git \
    -e vernemq_git_ref=v2.1.2

# Form cluster
ansible-playbook configure_cluster.yml

# Reset between scenarios
ansible-playbook restart_cluster.yml
```
